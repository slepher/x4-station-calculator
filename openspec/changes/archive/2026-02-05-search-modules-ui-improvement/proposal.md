# 模块搜索UI改进

## Why
当前搜索框的界面设计存在以下问题：
1. **遮挡问题**：搜索框弹出时会遮挡模块列表，用户无法预览模块列表的改动
2. **操作效率低**：需要先预览再添加，增加了操作步骤
3. **视觉干扰**：搜索框位置不合理，影响整体界面布局
4. **缺乏反馈**：模块添加和更新时缺乏清晰的视觉反馈

这些问题的存在降低了用户的使用效率和体验，需要通过界面优化来解决。

## What Changes
本次变更将实现以下改进：

### 界面布局优化
- 将搜索框从模块列表底部移动到顶部
- 实现搜索框从右侧弹出的效果，避免遮挡模块列表

### 交互流程简化  
- 取消搜索结果预览区，实现点击直接添加模块
- 优化弹出菜单行为，点击后保持菜单打开状态，支持连续添加

### 视觉反馈增强
- 添加模块添加时的边框高亮动画
- 实现模块数量更新时的数字闪烁动画
- 支持快速连续操作时的动画不冲突

### 技术架构改进
- 基于事件驱动的动画系统
- 智能ID对比避免删除模块导致的索引错位
- 使用Set数据结构支持多个动画同时显示

## 涉及的能力
- Vue 3组件开发
- CSS动画和过渡效果
- 响应式界面设计
- 用户交互优化

## 影响范围
- StationModuleList.vue：布局结构调整，动画监听逻辑
- StationModuleSelector.vue：搜索逻辑优化，菜单行为改进  
- StationModuleItem.vue：动画效果实现，样式优化
- 相关CSS样式文件：动画效果和布局调整
