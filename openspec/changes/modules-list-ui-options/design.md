# Module List UI Options Design

## Overview
为模块列表用户界面选项设计改进，包括规划区数量调整、劳动力加成选项移动和种族偏好选择功能。

## 功能设计

### 规划区数量调整选项
- 在用户规划区标题栏添加1/5, 1/3, 1/2, 2x, 3x, 5x选项按钮
- 实现点击选项时自动调整规划模块数量的功能
- 按钮风格同资源产出概览中的"每小时流量"标签保持一致
- 按钮没有选中状态, 点击一次执行一次对应操作
- 按钮组应该右对齐
- hover按钮颜色为琥珀色,同资源产出概览中的锁的颜色保持一致. 并且高亮
- 按钮的颜色同资源规划区这几个字的颜色保持一致
- 按钮有hover高亮功能
- 按钮切换逻辑是, 从xN切换到xM之后, 资源规划区的所有模块数量调整为当前数量xM/N向上取整
- 保持与现有手动输入功能的兼容性
- 规划区高度应该和之前保持一致
- 其他元素的样式应该和之前保持一致, 例如字体大小, 颜色, 间距等

### 劳动力加成选项位置调整
- 将计算劳动力加成选项从当前位置移动到自动工业区标题栏
- 将计算补给区劳动力加成选项移动到自动补给区标题栏
- 确保功能逻辑保持不变，仅调整UI位置
- 工业区高度应该和之前保持一致
- 其他元素的样式应该和之前保持一致, 例如字体大小, 颜色, 间距等

### 种族偏好选择下拉列表
- 在自动工业区标题栏添加种族选择下拉列表
- 下拉列表内容从consumption.json数据中获取可用种族
- 实现种族选择与生产计算逻辑的集成
- 补给区高度应该和之前保持一致
- 其他元素的样式应该和之前保持一致, 例如字体大小, 颜色, 间距等

## 技术实现
- 使用现有的Vue 3 + TypeScript架构
- 遵循Tailwind CSS样式规范
- 确保响应式设计和移动端兼容性
- 保持与现有数据模型和计算逻辑的一致性
